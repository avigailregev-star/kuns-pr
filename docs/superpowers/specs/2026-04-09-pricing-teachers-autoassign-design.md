## שלב 2

### A — ניהול לוח שעות מורים

#### מסד נתונים — טבלה חדשה `teachers` ב-Supabase

| עמודה | סוג | תיאור |
|---|---|---|
| `id` | uuid (PK) | מזהה ייחודי |
| `name` | text | שם המורה |
| `instrument_type` | text | קשת / נשיפה / פסנתר / שירה / אחר |
| `available_days` | text[] | מערך ימים: ["א","ב","ג",...] |
| `available_hours` | jsonb | `{"א": {"from":"15:00","to":"19:00"}, ...}` |
| `created_at` | timestamptz | תאריך יצירה |

#### API

- `GET /api/teachers` — רשימת כל המורים
- `POST /api/teachers` — הוספת מורה חדש
- `PUT /api/teachers/[id]` — עדכון מורה
- `DELETE /api/teachers/[id]` — מחיקת מורה

#### ממשק האדמין

לשונית חדשה **"מורים"** בדף `/admin`. מאפשרת:
- הצגת כל המורים בטבלה
- הוספת מורה חדש (שם, סוג כלי, ימים + שעות לכל יום)
- עריכה ומחיקה

**קבצים חדשים:**
- `app/api/teachers/route.js` — GET, POST
- `app/api/teachers/[id]/route.js` — PUT, DELETE
- `components/TeachersTab.jsx` — לשונית המורים
- `components/TeacherForm.jsx` — טופס הוספה/עריכה

**קבצים מושפעים:**
- `app/admin/(protected)/page.js` — הוספת לשונית מורים
- `app/admin/(protected)/layout.js` — ניווט אם רלוונטי

### B — שיבוץ אוטומטי לתלמידים ממשיכים

**מתי מופעל:** כשאדמין פותח רשומה מסוג `continue` (ממשיך) בטבלת הניהול.

#### לוגיקת הצעות

**תזמורת/מקהלה** — לפי `instrument_type` של המורה המשובץ:
| סוג כלי | שיבוץ מוצע |
|---|---|
| קשת | תזמורת כלי קשת |
| נשיפה | תזמורת כלי נשיפה |
| פסנתר | מקהלה |
| שירה | מקהלה |

**תיאוריה** — ביום זהה ל-`assigned_day` של הרשומה (עדיפות ראשונה).

**מורה מוצע** — מורה שיש לו זמינות ביום `assigned_day` (לפי `available_days` בטבלת `teachers`).

#### תצוגה ב-AdminTable

מתחת לשדות השיבוץ ברשומת ממשיך מוצגים badges ירוקים:
- "מוצע: תזמורת כלי קשת"
- "מוצע: תיאוריה ביום ג׳"
- "מורה פנוי ביום זה: [שם]"

לחיצה על badge ממלאת את השדה הרלוונטי. האדמין יכול לשנות ידנית.

**קבצים מושפעים:**
- `components/AdminTable.jsx` — הצגת הצעות שיבוץ לרשומות `continue`
